# Phase 02: Scope MVP and Roadmap

## Objective
Translate discovery into executable scope with clear milestone boundaries.

## Core Activities
- Define MVP versus later releases.
- Split roadmap into milestones (M1, M2, M3).
- Lock critical must-have flows:
  - Authentication
  - Tenant management
  - Course/content delivery
  - Assessments/exams
  - Payment/billing
- Build risk register focused on:
  - Multi-tenancy
  - Offline behavior
  - Video/content delivery via CDN
  - Compliance and data privacy

## Inputs
- Phase 01 outputs.
- Technical feasibility assumptions.

## Deliverables
- MVP scope statement.
- Milestone roadmap with dates and owners.
- Risk register with mitigation and contingency owner.

## Exit Criteria
- MVP stories prioritized and estimated.
- Risks classified (high/medium/low) with actions.
- Stakeholder sign-off for execution start.

## File Outputs
- `templates/mvp-roadmap-template.md`
- `templates/risk-register-template.md`
